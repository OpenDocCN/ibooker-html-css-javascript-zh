## 同构 Web 应用的最佳实践

| 最佳实践 | 部分 |
| --- | --- |
| --- | --- |
| 设置 webpack 以准备生产使用。5.5.2 |
| 构建可组合的 React 组件，提高代码的可重用性。4.3.1, 4.3.2 |
| 使用扩展运算符(...)模式确保在使用 Redux 时你的状态是不可变的。6.3.1 |
| 使用 React Router 为服务器实现路由。7.2.1 |
| 使用 React 的 renderToString 在服务器上渲染组件。7.2.3 |
| 使用静态函数来指示应用中每个容器组件所需的数据。7.3.2 |
| 序列化在服务器上获取的数据，并将其附加到 DOM 上。8.2.1 |
| 解析 DOM 中提供的数据，并使用它来初始化应用的状态。8.2.2 |
| 通过使用 React 的生命周期方法 componentDidMount 来避免同构渲染警告。8.3.3 |
| 确保你在所有将运行代码的环境中测试你的代码（服务器和浏览器）。9.1.1, 9.2.1, 9.2.2 |
| 使用环境变量来隔离只能在浏览器中运行的代码。Webpack 允许你将它们注入到你的浏览器代码中（process.env.BROWSER）。10.1.2, 10.1.3 |
| 为了管理 SEO，使用静态函数向服务器上 HTML 页面的<head>部分添加元标签。10.2.1, 10.2.2 |
| 以同构方式基于 cookie 处理用户会话。11.4.1 |
