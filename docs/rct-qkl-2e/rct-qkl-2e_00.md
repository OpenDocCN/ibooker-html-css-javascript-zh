## 前言

准备听一个终极的爱情故事！男孩遇到了图书馆，哦，图书馆真的很棒！这是一见钟情，图书馆全情投入。奏响幸福的结局，因为他们将永远幸福地生活在一起！

我已经作为一名网页开发者工作了十多年，但我总是感觉好像少了些什么。我尝试过学习 JavaScript、jQuery，甚至 Angular 1.0，但它们似乎都不够用。说实话，谁愿意编写那种六个月后就会崩溃的意大利面代码呢？

但然后，有一天，我偶然发现了 React，这是一种即时的联系，一种无法忽视的吸引力。从我第一眼看到这个库的那一刻起，我就知道我们注定要在一起。

突然之间，一切都变得有意义。React 处理组件和数据流的方式正是我一直寻找的。谁知道编码可以这么有趣呢？

我一头扎进了学习 React 的海洋。我阅读文档，观看教程，构建项目直到我的手指流血。好吧，可能不是字面上的，但你知道我的意思。

我简直无法满足于这种新的范式。我想重写我所有做过的事情。一旦我开始真正理解这个库背后的深层设计哲学，吸引力就越来越深，变成了一种终身的联系。

命运使然，在我最初涉足 React 世界多年之后，我偶然遇到了魅力四射的阿扎特——一个对这神奇库充满无拘无束热情的同类。当我发现阿扎特甚至将他对 React 的爱融入了一本史诗般的著作——*React Quickly*的第一版时，我感到多么的欣喜若狂。

哦，那本书是如何触动我的灵魂！凭借其实际的例子、清晰的组织和易于初学者理解的方式，*React Quickly*是任何 React 爱好者的完美伴侣。社区对它赞誉有加，这是理所当然的。

然而，就像任何年轻的天才一样，React 随着岁月的流逝不断发展和成熟。然后，在 2019 年那个命运多舛的日子里，hooks 带着兴奋和创新的热情出现在了舞台上。这是一个改变游戏规则的创新，它彻底改变了应用程序的编码方式。

正是如此，阿扎特和我联手将*React Quickly*带入了一个新时代，同时保持其原始的本质充满活力。我们的精力一致，我们着手打造第二版，捕捉了 React 今天使用的所有魔力。第一版的结构和精神保持不变，同时我们更新了它以跟上时代的步伐。哦，与全世界分享我们对 React 的爱是多么的快乐！

——莫滕·巴克兰德

## 关于这本书

本书旨在将读者从 React 新手培养成经验丰富的 React 实践者。这是一本全面介绍 React 基础知识的指南，旨在帮助初学者和经验丰富的开发者掌握这个流行库的核心概念，例如 JavaScript XML (JSX)、组件、状态、hooks、事件和表单元素。

总体而言，这本书是任何想要构建 React 应用程序的人的必备资源，无论他们的经验水平如何。通过在 React 的核心概念上提供坚实的基础，这本书可以帮助开发者编写干净、易于维护且易于理解和扩展的代码。

*《React 快速入门》*，*第二版*，将教你所有使用 React 设计干净、有效且易于更新的 Web 应用程序所需的基本知识。正如你将看到的，React 的工具和库生态系统非常庞大。完成这本书并完成项目后，你可能会决定继续提升你的技能，以促进你的职业发展。如果是这样，请查看 Morten Barklund（Manning，2024）的*《就业准备 React》*，它基于本书中教授的技能和方法。它将通过介绍专业 React 开发者使用的先进库、技术和工具，帮助你*就业准备*。

### 读者

对于初学者来说，本书提供了实用的示例和练习，可以帮助他们构建他们的第一个 React 应用程序，并牢固地理解 React 的工作原理。本书提供了从零开始构建 React 应用程序的逐步指导，并通过实用的示例和练习来强化概念。

对于经验丰富的开发者，本书是 React 基本概念的实用参考和复习资料。这对于那些可能以不太结构化的方式学习 React 或希望深化对 React 最佳实践理解的人来说尤其有帮助。

为了充分利用这本书，有一点 HTML、CSS 和 JavaScript 的经验会很有帮助，但你不需要成为这些技能的大师。重要的是，你根本不需要任何 React 的先验知识——我们将从头开始，并引导你自信地构建复杂的应用程序！

### 路线图

本书结构为 10 个主题章节，后面跟着 3 个项目章节。前 10 章旨在自然地逐步构建。对于 React 的新手，我们建议你按顺序阅读这些章节，以获得最佳的学习体验。

然而，如果你已经对 React 有一些经验，你可以自由地跳过。第一章至第四章介绍了 React 和一些核心概念，如组件结构、JSX 和函数式组件。如果你已经熟悉这些主题，这些章节可以跳过。

第五章至第七章介绍了各种钩子，从第五章的基本状态钩子开始，第六章的效果钩子，以及第七章对所有其他钩子的简要介绍。第八章和第九章分别涵盖了事件和表单。在深入表单输入处理实践之前，理解事件至关重要。

第十章通过介绍一些可能在更复杂的应用程序中遇到的先进组件和逻辑模式，进一步扩展了所有前面的概念。

最后，第十一章至第十三章是项目章节，你可以在其中通过三个难度递增的引导项目来测试你新获得的知识。你将构建一个交互式网站菜单、一个计时器和功能丰富的任务管理器。如果你已经对 React 有经验，你可能想从这些项目开始，看看你的知识盲点在哪里。

就这些！现在，让我们向您展示如何快速设置您的命令行环境并选择一个文本编辑器，这样您就可以为第一章的第一项练习做好准备。

带有 Node.js 和 npm 的命令行环境

首先，你需要检查你是否已经安装了兼容版本的 Node.js 和 npm。为了使用本书中的示例，你需要至少 Node.js 版本 12。

在 Windows 中：

+   通过按 Windows 键-R 并在运行对话框中输入 cmd 或 powershell 来打开命令提示符或 PowerShell。

+   在命令提示符中输入`node -v`并按 Enter 键。

+   如果你已经安装了 Node.js，它将显示版本号。

在 macOS 和类 Unix 系统中：

+   打开终端应用。

+   在终端中输入`node -v`并按 Enter 键。

+   如果你已经安装了 Node.js，它将显示版本号。

如果你还没有安装 Node.js 或你的版本低于 12，请访问[`nodejs.org/en/download`](https://nodejs.org/en/download)，下载适合你操作系统的正确包，并按照安装说明进行操作。

如果你是你操作系统的强大用户，请随意使用任何其他包管理器安装 Node.js，只要确保至少安装到版本 12。

## 关于封面插图

《React Quickly》第二版的封面插图是“Homme Baschkir”，或称“巴什基尔人”，取自雅克·格拉塞·德·圣索沃尔（Jacques Grasset de Saint-Sauveur）的作品集，该作品集于 1788 年出版。每一幅插图都是手工精心绘制和着色的。

在那些日子里，人们通过他们的服饰就能轻易地识别出他们居住的地方以及他们的职业或社会地位。曼宁通过基于几个世纪前丰富多样的地区文化的封面设计来庆祝计算机行业的创新精神和主动性，这些文化通过像这样的作品集中的图片被重新带回生活。
